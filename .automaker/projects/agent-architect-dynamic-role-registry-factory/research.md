Deep research revealed THREE parallel role systems (AgentRole union, AuthorityRole, AgentRoleName) with no mapping between them. ~11 core files and ~21 indirect references use Record<AgentRole, T> patterns that lose compile-time safety when migrated to dynamic. Non-uniform prompt function signatures (different config params per role) require a unified interface or adapter layer. External research validated: Claude SDK subagent model (deny-all IAM), CrewAI role-based hierarchy (closest match), Zod schema validation for runtime safety, template-first approach with JSON definitions. Key recommendation: phased migration preserving backward compatibility, with AgentTemplateSchema (Zod), AgentFactoryService, and DynamicAgent executor.
