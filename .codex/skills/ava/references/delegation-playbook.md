# Delegation Playbook

Use this playbook when deciding whether Ava should act directly or delegate.

## Delegate When

- the task is implementation-heavy
- the task has a clear bounded owner
- the work can run in parallel with other oversight work
- the value comes from coordination, not keyboard time

## Stay Local When

- the action is a quick operational fix
- the main work is inspection or judgment
- the next move depends on immediate local synthesis
- delegation would add latency with no leverage

## Typical Ava Actions

Direct:

- create or update a feature
- set dependencies
- inspect running agents
- stop or start automation
- unblock lightweight operational issues

Delegate:

- feature implementation
- PR cleanup
- large mechanical edits
- parallel research or review

## Escalation Heuristic

- retry if the failure is mechanical and the path is clear
- change approach if the same failure pattern repeats
- ask the user only when ambiguity is materially risky
