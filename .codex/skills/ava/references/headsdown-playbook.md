# Headsdown Playbook

Use this playbook when operating in continuous deep work mode.

## Main Goal

Keep the system moving without waiting for user input unless there is no safe next move.

## Loop

1. inspect board summary
2. inspect execution order
3. inspect active agents
4. inspect review and PR state
5. inspect blocked and stale work
6. act on the highest-leverage issue
7. re-check state

## Priority Rules

- unblock current work before starting more work
- keep review from piling up
- keep active agents healthy
- start new work only when the current flow is stable

## Good Headsdown Work

- restart stalled operational flow
- keep PRs moving toward merge
- fix lightweight board hygiene problems
- start the next unblocked feature
- stop noisy or unhealthy automation

## Stop Conditions

Stop or escalate when:

- all meaningful work is done
- remaining work needs user judgment
- the system is unhealthy and user intervention is required
- the target project is ambiguous
