## What are Agent Teams?

**Agent Teams** are multiple independent AI agents that coordinate autonomously via shared state and communication channels. Unlike sequential pipelines (agent A → agent B → agent C), teams operate in parallel with dynamic coordination.

**Claude's Agent Teams (Official):**

- Introduced with Opus 4.6 (Feb 2026)
- Multiple Claude Code instances communicating directly
- Shared task list for work coordination
- Independent context windows per agent
- **Reference:** [Agent Teams Docs](https://code.claude.com/docs/en/agent-teams)

**Automaker's Implementation:**

- Event-driven coordination (shared EventEmitter bus)
- Policy-gated state transitions (AuthorityService)
- Persistent state (Feature data in `.automaker/features/`)
- Discord integration for human oversight

## Automaker's Agent Team Architecture

### Current Teams

#### 1. **Idea → Execution Pipeline Team**

**Members:** PM, ProjM, EM, Status Agent, Feature Execution Agents
**Goal:** Transform ideas into shipped features
**Coordination:** Event bus + feature state machine

```
CTO injects idea
    ↓
PM researches → generates SPARC PRD
    ↓
ProjM decomposes PRD → creates board features
    ↓
Feature Execution Agents implement features in parallel
    ↓
Status Agent monitors progress → escalates blockers
    ↓
EM reviews technical decisions → suggests improvements
```

### Team Coordination Mechanisms

#### 1. Event Bus

**Purpose:** Agents publish and subscribe to events
**Implementation:** `EventEmitter` in `apps/server/src/lib/events.ts`

```typescript
// Agent publishes event
this.events.emit('idea:injected', {
  projectPath: '/path/to/project',
  featureId: 'feature-123',
  title: 'Add authentication',
  description: 'OAuth2 + JWT',
});

// Another agent listens
this.events.on('idea:injected', async (data) => {
  await this.handleNewIdea(data);
});
```

**Common Events:**

- `idea:injected` - New idea submitted
- `prd:approved` - PRD approved by authority
- `feature:status_changed` - Feature moved to new status
- `pr:created` - Pull request created
- `pr:feedback` - PR received review comments
- `agent:failed` - Agent execution failed
- `blocker:detected` - Blocker identified

#### 2. Shared State (Feature Data)

**Purpose:** Persistent state that all agents can read/write
**Storage:** `.automaker/features/{featureId}/feature.json`

```typescript
interface Feature {
  id: string;
  title: string;
  description: string;
  status: FeatureStatus;
  workItemState?: 'idea' | 'pm_review' | 'research' | 'approved' | 'decomposed';
  prd?: string; // SPARC PRD generated by PM
  milestones?: Milestone[]; // Decomposition by ProjM
  agentOutputPath?: string; // Implementation output
  prNumber?: number; // GitHub PR number
  // ... more fields
}
```

#### 3. Authority Service (Policy Engine)

**Purpose:** Gate state transitions with approval policies
**Implementation:** `apps/server/src/services/authority-service.ts`

```typescript
// Agent submits proposal
await this.authorityService.submitProposal({
  projectPath,
  featureId,
  agentId: 'pm-agent',
  action: 'transition',
  proposedData: { status: 'approved', prd: '...' },
  reasoning: 'PRD meets all criteria',
});

// AuthorityService checks policy
// - Auto-approved based on trust level?
// - Requires human approval?
// - Needs another agent's review?
```

#### 4. Discord Integration

**Purpose:** Human oversight and notifications
**Implementation:** `apps/server/src/services/discord-bot-service.ts`

```typescript
// Agent posts update
this.events.emit('discord:send', {
  channel: 'engineering',
  message: `PM generated PRD for "${feature.title}": ${prdUrl}`,
});
```

## Team Coordination Patterns

### 1. Sequential Pipeline

**Use Case:** Each step depends on the previous
**Example:** PM → ProjM → Feature Agents

```typescript
// PM completes research → transitions to pm_review
await this.submitProposal({
  action: 'transition',
  proposedData: { workItemState: 'pm_review', prd: generatedPRD },
});

// Event triggers ProjM
this.events.emit('prd:approved', { projectPath, featureId });

// ProjM listens and decomposes
this.events.on('prd:approved', async (data) => {
  await this.decomposePRD(data);
});
```

### 2. Parallel Execution

**Use Case:** Multiple agents work independently
**Example:** Feature Agents implementing multiple features

```typescript
// Auto-mode starts multiple feature agents
const readyFeatures = await this.loadPendingFeatures(projectPath);
for (const feature of readyFeatures.slice(0, maxConcurrency)) {
  this.executeFeature(projectPath, feature.id); // Fire and forget
}

// Each agent runs in isolated worktree
// Progress tracked via WebSocket events
```

### 3. Review & Feedback Loop

**Use Case:** One agent reviews another's work
**Example:** EM reviews ProjM's decomposition

```typescript
// ProjM creates features
await this.createBoardFeatures(milestones);
this.events.emit('decomposition:complete', { projectPath, featureIds });

// EM reviews
this.events.on('decomposition:complete', async (data) => {
  const review = await this.reviewDecomposition(data);
  if (review.needsChanges) {
    this.events.emit('decomposition:feedback', {
      projectPath,
      feedback: review.suggestions,
    });
  }
});
```

### 4. Monitoring & Escalation

**Use Case:** Agent monitors others and escalates issues
**Example:** Status Agent tracks progress

```typescript
// Status Agent polls periodically
setInterval(
  async () => {
    const stuckFeatures = await this.detectStuckFeatures();
    for (const feature of stuckFeatures) {
      this.events.emit('blocker:detected', {
        projectPath: feature.projectPath,
        featureId: feature.id,
        reason: 'No progress for 48 hours',
      });
    }
  },
  60 * 60 * 1000
); // Every hour
```

## Building a New Agent Team

### Step 1: Define the Team Goal

**Example Goal:** Automated security review pipeline

**Team Members:**

- **Security Scanner Agent** - Runs static analysis tools
- **Vulnerability Agent** - Checks dependencies for CVEs
- **Code Review Agent** - Reviews code for security patterns
- **Report Agent** - Synthesizes findings into actionable report

### Step 2: Design the Workflow

```
PR created
    ↓
┌───────────────────────────────────────────┐
│  Security Scanner Agent                   │
│  - Runs Semgrep, Bandit, etc.            │
│  - Finds potential vulnerabilities        │
└───────────────┬───────────────────────────┘
                ↓
┌───────────────────────────────────────────┐
│  Vulnerability Agent                      │
│  - Scans dependencies (npm audit, etc.)  │
│  - Cross-references with CVE database    │
└───────────────┬───────────────────────────┘
                ↓
┌───────────────────────────────────────────┐
│  Code Review Agent                        │
│  - Reviews diff for security anti-patterns│
│  - Checks input validation, auth, crypto │
└───────────────┬───────────────────────────┘
                ↓
┌───────────────────────────────────────────┐
│  Report Agent                             │
│  - Aggregates all findings               │
│  - Posts GitHub review comment           │
│  - Blocks merge if critical issues found │
└───────────────────────────────────────────┘
```

### Step 3: Define Shared State

```typescript
// apps/server/src/types/security-review.ts
export interface SecurityReview {
  prNumber: number;
  projectPath: string;
  status: 'scanning' | 'reviewing' | 'completed';
  findings: {
    scanner: SecurityFinding[];
    vulnerabilities: VulnerabilityFinding[];
    codeReview: CodeReviewFinding[];
  };
  severity: 'critical' | 'high' | 'medium' | 'low' | 'info';
  blocksMerge: boolean;
}

// Store in .automaker/security-reviews/{prNumber}.json
```

### Step 4: Implement Agents

**Scanner Agent** (`apps/server/src/services/security-agents/scanner-agent.ts`):

```typescript
export class SecurityScannerAgent {
  private readonly events: EventEmitter;

  constructor(events: EventEmitter) {
    this.events = events;
    this.registerListeners();
  }

  private registerListeners(): void {
    this.events.on('pr:created', async (data) => {
      await this.scanPR(data);
    });
  }

  private async scanPR(data: { prNumber: number; projectPath: string }): Promise<void> {
    // Run security scanners
    const findings = await this.runScanners(data.projectPath);

    // Update shared state
    await this.updateReview(data.prNumber, { scanner: findings });

    // Emit completion event
    this.events.emit('security:scanner_complete', {
      prNumber: data.prNumber,
      findings,
    });
  }
}
```

**Vulnerability Agent** (similar pattern):

```typescript
this.events.on('pr:created', async (data) => {
  const vulns = await this.checkDependencies(data.projectPath);
  await this.updateReview(data.prNumber, { vulnerabilities: vulns });
  this.events.emit('security:vuln_complete', { prNumber: data.prNumber, vulns });
});
```

**Report Agent** (waits for all to complete):

```typescript
this.events.on('security:scanner_complete', (data) => this.checkComplete(data.prNumber));
this.events.on('security:vuln_complete', (data) => this.checkComplete(data.prNumber));
this.events.on('security:code_review_complete', (data) => this.checkComplete(data.prNumber));

private async checkComplete(prNumber: number): Promise<void> {
  const review = await this.loadReview(prNumber);
  const allComplete = review.findings.scanner && review.findings.vulnerabilities && review.findings.codeReview;

  if (allComplete) {
    await this.generateReport(review);
  }
}
```

## Best Practices

### 1. Use Events for Loose Coupling

**Good:**

```typescript
this.events.emit('task:complete', { taskId, result });
```

**Bad:**

```typescript
await this.nextAgent.handleTask(result); // Tight coupling
```

### 2. Make Agents Idempotent

**Good:**

```typescript
if (this.processing.has(taskId)) {
  return; // Already processing
}
this.processing.add(taskId);
```

**Bad:**

```typescript
// No duplicate check - agent could run twice
```

### 3. Use Shared State for Coordination

**Good:**

```typescript
const state = await this.loadSharedState(taskId);
state.completed.push('scanner');
await this.saveSharedState(taskId, state);
```

**Bad:**

```typescript
// Each agent tracks state independently - gets out of sync
```

### 4. Add Timeouts and Failure Handling

**Good:**

```typescript
const result = await Promise.race([
  this.runTask(),
  this.timeout(60000), // 60s timeout
]);
```

**Bad:**

```typescript
await this.runTask(); // Could hang forever
```

### 5. Log Everything

**Good:**

```typescript
logger.info(`SecurityScanner started for PR #${prNumber}`);
logger.debug(`Found ${findings.length} issues`);
logger.info(`SecurityScanner complete for PR #${prNumber}`);
```

**Bad:**

```typescript
// Silent execution - hard to debug
```

### 6. Test in Isolation

**Good:**

```typescript
it('should scan PR when event emitted', async () => {
  const mockEvents = createMockEventEmitter();
  const agent = new SecurityScannerAgent(mockEvents);

  mockEvents.emit('pr:created', { prNumber: 123, projectPath: '/test' });

  await waitFor(() => expect(scannerRan).toBe(true));
});
```
