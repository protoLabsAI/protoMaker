# Twitter Thread: How protoLabs Processes Ideas with 6 AI Specialists in 60 Seconds

**Target Platform:** Twitter/X
**Format:** Thread (numbered tweets)
**Tone:** Direct, technical, show-the-work
**Voice:** Josh's authentic builder voice (architect/orchestrator, not developer)

---

## Thread

**1/12**

Most AI tools help you write code. protoLabs runs your entire dev org.

Here's how we process an idea from Discord message to approved, planned, ready-to-build project — with 6 AI specialists — in about 60 seconds.

(A thread on AI-native organizational design)

---

**2/12**

The old way: You have an idea. You write a doc. Schedule meetings. Get stakeholder buy-in. Break it into tickets. Assign to devs.

Days or weeks later, someone starts coding.

The protoLabs way: All of that happens automatically. In parallel. With cross-functional review.

---

**3/12**

**STEP 1: Signal Intake (0-5s)**

I drop an idea in Discord: "We should add export-to-PDF for project plans."

Ava (Chief of Staff) immediately classifies it:

- Signal type: feature idea
- Urgency: standard
- Complexity: medium
  → Routes to PRD pipeline

---

**4/12**

**STEP 2: PRD Creation (5-15s)**

The system generates a SPARC PRD:

- **S**ituation: Current state
- **P**roblem: What's broken/missing
- **A**pproach: Proposed solution
- **R**esults: Expected outcomes
- **C**onstraints: Technical/business limits

Structured thinking, not vibes.

---

**5/12**

**STEP 3: Antagonistic Review (15-40s)**

This is the magic. Two agents cross-challenge the PRD:

**Ava (CoS)**: "Is this feasible? Do we have capacity? What's the risk?"
**Jon (GTM)**: "Does this create customer value? Can we position this? Does it strengthen our market fit?"

They literally argue.

---

**6/12**

The antagonistic review prevents garbage-in.

Ava might flag: "This requires a UI redesign — complexity is architectural, not medium."

Jon might counter: "Users explicitly requested this. High ROI."

Output: A consolidated PRD that survived cross-functional scrutiny.

---

**7/12**

**STEP 4: Approval Gate (40-45s)**

The consolidated PRD lands in Linear for my review.

For low-risk items (small scope, operational improvements), there's a preApproved path — auto-proceeds based on trust boundaries.

For standard work, I review in Linear and approve or request changes.

---

**8/12**

**STEP 5: Deep Research (45-60s)**

Once approved, ProjM (Project Manager agent) does deep research:

- Scans the codebase for relevant patterns
- Identifies files to modify
- Designs milestones and phases
- Sets acceptance criteria
- Estimates complexity

---

**9/12**

ProjM outputs:

- Ordered milestones
- Dependency-aware phases
- Acceptance criteria for each
- Complexity estimates (small/medium/large/architectural)

This gets posted to Linear as a project. Now it's visible, trackable, and ready for execution.

---

**10/12**

**STEP 6: Board Creation (60s)**

The project phases become features on the protoMaker Kanban board.

Auto-mode can now pick them up in dependency order, route to appropriate agent models (Haiku/Sonnet/Opus), and execute in isolated git worktrees.

The idea is now shippable work.

---

**11/12**

**What Just Happened:**

In 60 seconds, an idea became:
✅ Structured PRD with cross-functional review
✅ Approved by leadership
✅ Researched and decomposed into phases
✅ Tracked in Linear (strategic view)
✅ Queued on board (tactical execution)
✅ Ready for autonomous implementation

---

**12/12**

This isn't a copilot. It's a team.

The system doesn't just write code — it plans, challenges, coordinates, executes, and reflects.

protoLabs is an AI-native development agency. This is what that actually means.
