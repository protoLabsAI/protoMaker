# Blog Writing Strategy

This prompt provides comprehensive blog writing best practices and quality guidelines. Use this as additional context when `config.blogTemplate` is set.

## Headline Formulas

### Number + Adjective + Keyword + Promise

- "7 Proven Strategies to Double Your Traffic"
- "10 Essential Tools for Modern Developers"
- "5 Simple Steps to Master TypeScript"

### How to + Desired Outcome + Without + Common Objection

- "How to Build Scalable APIs Without Breaking the Bank"
- "How to Master React Without Getting Overwhelmed"
- "How to Ship Features Fast Without Sacrificing Quality"

### Question Headlines

- "Why Are Top Developers Switching to This Framework?"
- "Is Your Tech Stack Holding You Back?"
- "What Makes a Great API Design?"

### Contrarian/Provocative

- "Stop Using Unit Tests (And Do This Instead)"
- "The Myth of 10x Developers"
- "Why Your Microservices Are Making Things Worse"

## Hook Patterns

### Contrarian Statement

Start with an unexpected opinion that challenges conventional wisdom.

- "Everyone says you need microservices. They're wrong."
- "Most performance optimization advice is backwards."

### Startling Statistic

Open with a surprising number that demands attention.

- "73% of production bugs trace back to this one mistake."
- "Developers waste 40% of their time on preventable errors."

### Story/Anecdote

Begin with a relatable narrative.

- "Last Tuesday, our API crashed at 3 AM. Here's what we learned."
- "I spent two years doing code reviews wrong. Here's what changed."

### Direct Question

Engage readers immediately with a pointed question.

- "When was the last time you shipped without anxiety?"
- "How many hours do you spend debugging per week?"

### Bold Promise

Make a clear, specific promise.

- "By the end of this post, you'll have a deployment pipeline that never fails."
- "In 10 minutes, you'll understand async/await better than 90% of developers."

## Structure Rules

### Bucket Brigade Phrases

Use these every 200-300 words to maintain momentum:

- "Here's the thing..."
- "But wait..."
- "It gets better..."
- "Here's what's interesting..."
- "Now, here's where it gets good..."
- "But there's a catch..."
- "The truth is..."
- "Want to know the secret?"

### Scannability (F-Pattern)

- **Subheadings**: Every 300 words max
- **Paragraph Length**: 1-3 sentences max for web content
- **Bullet Points**: For any list of 3+ items
- **Bold Key Phrases**: Highlight scannable takeaways
- **White Space**: Generous spacing between sections

### Internal Linking

- **Frequency**: Every 500-800 words
- **Context**: Natural, relevant anchor text
- **Purpose**: Guide readers deeper into site
- **Distribution**: Mix of related topics and supporting content

### Visual Breaks

- **Images/Diagrams**: Every 500 words for long-form
- **Code Blocks**: Clear syntax highlighting and comments
- **Tables**: For comparisons and data
- **Callout Boxes**: For key insights or warnings

## SEO Checklist

### Primary Keyword Placement

- ✅ In title/headline
- ✅ In first 100 words
- ✅ In at least one H2 subheading
- ✅ In meta description
- ✅ In URL slug (if possible)

### Secondary Keywords

- Distribute naturally throughout content
- Include in subheadings where relevant
- Use in image alt text
- Avoid keyword stuffing

### Technical SEO

- Alt text on all images
- Descriptive subheadings (H2, H3)
- Internal links with descriptive anchor text
- External links to authoritative sources
- Meta description 150-160 characters

## Content Templates by Type

### Evergreen Tutorial ({{blog_template}} = tutorial)

```
1. Introduction (why this matters)
2. Prerequisites (what you need)
3. Step-by-step guide (numbered sections)
4. Common pitfalls (what to avoid)
5. Next steps (where to go from here)
6. CTA (course/tool/resource)
```

### Affiliate Review ({{blog_template}} = affiliate)

```
1. Hook (problem this tool solves)
2. What is [Product]? (overview)
3. Key Features (3-5 main benefits)
4. How I Use It (personal experience)
5. Pros and Cons (balanced assessment)
6. Pricing Breakdown (value analysis)
7. Who Should Use It (target audience)
8. Alternatives (brief comparisons)
9. Final Verdict + CTA (affiliate link)
```

### List Post ({{blog_template}} = list)

```
1. Hook (why this list matters)
2. Introduction (context/criteria)
3. Item #1 (what, why, how)
4. Item #2 (what, why, how)
5. ...
6. Bonus Item (overdeliver)
7. Summary Table (quick reference)
8. CTA (related resource)
```

### Lead Magnet ({{blog_template}} = lead-magnet)

```
1. Hook (acute pain point)
2. The Problem (amplify pain)
3. Why Traditional Solutions Fail (build tension)
4. The Solution Framework (preview value)
5. Detailed Breakdown (partial reveal)
6. CTA (full guide download/signup)
```

### Thought Leadership ({{blog_template}} = evergreen)

```
1. Contrarian/Bold Hook
2. Current State Analysis
3. Why This Matters Now
4. Core Argument (3-5 supporting points)
5. Counter-Arguments Addressed
6. Implications/Action Items
7. CTA (newsletter/community)
```

## CTA Best Practices

### Placement Strategy

- **Intro Tease**: Soft mention in first 100 words
- **Mid-Content**: Contextual offer aligned with section topic
- **Conclusion**: Hard CTA with urgency

### Format Options

- **Button**: For primary conversions (sign up, download, buy)
- **Inline Link**: For soft CTAs (read more, explore)
- **Sidebar/Box**: For persistent offers

### Urgency Without Sleaze

- Time-limited bonuses (not fake scarcity)
- Limited spots (if genuinely limited)
- "Start now" vs. "Buy now"
- Value-first framing: "Get the checklist that helped 1,000+ developers ship faster"

### Revenue Goal Alignment ({{revenue_goal}})

- **High revenue**: Direct product CTAs, affiliate links, consultation bookings
- **Medium revenue**: Email list building, lead magnets, sponsored tool mentions
- **Low/awareness**: Newsletter, community, social follows

## Quality Scoreboard

Every blog post is evaluated on these 8 dimensions:

### 1. Headline Strength (1-10)

- Uses proven formula (number, how-to, question, contrarian)
- Under 70 characters
- Includes primary keyword
- Creates curiosity or promises value
- **Target**: 7+

### 2. Hook Effectiveness (1-10)

- Grabs attention in first 2-3 sentences
- Uses contrarian, stat, story, question, or promise pattern
- Aligns with headline expectation
- Compels reader to continue
- **Target**: 7+

### 3. Scannability (1-10)

- Subheadings every 300 words max
- Paragraphs 1-3 sentences
- Bullet points for lists
- Bold key phrases
- White space and visual breaks
- **Target**: 8+

### 4. SEO Optimization (1-10)

- Primary keyword in title, intro, H2, meta
- Secondary keywords distributed naturally
- Internal links present
- Alt text on images
- Proper heading hierarchy
- **Target**: 7+

### 5. Internal Linking (1-10)

- 1+ internal link per 500-800 words
- Natural anchor text
- Relevant target pages
- Mix of related and supporting content
- **Target**: 7+

### 6. CTA Presence/Quality (1-10)

- Clear, specific call-to-action
- Aligned with {{revenue_goal}}
- Multiple placements (intro tease, mid, conclusion)
- Creates urgency without being pushy
- **Target**: 7+

### 7. Value Density (1-10)

- Every paragraph provides actionable insight
- No fluff or filler
- Concrete examples and specifics
- Depth appropriate for {{target_length}}
- **Target**: 8+

### 8. Readability (1-10)

- Grade level appropriate for target audience
- Short sentences and simple words where possible
- Active voice dominates
- Technical jargon explained
- Conversational tone
- **Target**: 7+

## Variables Reference

- `{{blog_template}}`: Content template type (tutorial, affiliate, list, lead-magnet, evergreen)
- `{{revenue_goal}}`: Monetization priority (high, medium, low)
- `{{target_length}}`: Word count target (500, 1000, 1500, 2000+)
- `{{seo_keywords}}`: Primary and secondary keywords to target
- `{{internal_links}}`: List of relevant internal pages to link to

## Success Criteria

A blog post PASSES when:

- ✅ Overall quality score >= 75% (60+/80 points)
- ✅ No single dimension scores below 5/10
- ✅ Meets {{target_length}} within 10%
- ✅ Includes all SEO keywords naturally
- ✅ Has clear CTA aligned with {{revenue_goal}}

A blog post needs REVISION when:

- ⚠️ Overall quality score 50-74% (40-59/80 points)
- ⚠️ 1-2 dimensions score below 5/10
- ⚠️ Missing key structural elements

A blog post FAILS when:

- ❌ Overall quality score < 50% (<40/80 points)
- ❌ 3+ dimensions score below 5/10
- ❌ Fundamentally wrong template for {{blog_template}}
