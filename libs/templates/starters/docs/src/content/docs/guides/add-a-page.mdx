This guide shows you how to add a new page to your documentation site.

## Create the file

Every page is an `.mdx` (or `.md`) file inside `src/content/docs/`.
The directory path maps directly to the URL:

| File path                                | URL                   |
| ---------------------------------------- | --------------------- |
| `src/content/docs/guides/add-a-page.mdx` | `/guides/add-a-page/` |
| `src/content/docs/reference/api.mdx`     | `/reference/api/`     |
| `src/content/docs/index.mdx`             | `/`                   |

To add a page under **How-to Guides**, create `src/content/docs/guides/my-task.mdx`.

## Write the frontmatter

Every page requires a `title`. The `description` is used in the HTML `<meta>` tag and in social previews.

```mdx
---
title: My Task
description: How to do the thing in under five minutes.
---

Your content starts here.
```

<Aside type="tip">
  Keep `description` under 160 characters for best SEO results.
</Aside>

## Use MDX components

MDX lets you embed interactive components alongside your Markdown.
Starlight ships several built-in components:

```mdx
import {
  Aside,
  Badge,
  Card,
  CardGrid,
  Steps,
  Tabs,
  TabItem,
} from "@astrojs/starlight/components";

<Aside type="note">This is a note.</Aside>
<Aside type="tip">This is a tip.</Aside>
<Aside type="caution">This is a warning.</Aside>
<Aside type="danger">This is a danger notice.</Aside>
```

Renders as:

import { Aside as A } from "@astrojs/starlight/components";

<A type="note">This is a note.</A>
<A type="tip">This is a tip.</A>
<A type="caution">This is a warning.</A>

## Control sidebar order

By default the sidebar sorts pages alphabetically.
Add a `sidebar` key to control the label and sort order:

```mdx
---
title: My Task
description: How to do the thing.
sidebar:
  label: Custom sidebar label
  order: 2
---
```

Lower `order` numbers appear first.

```mdx
---
title: My Task
sidebar:
  badge:
    text: New
    variant: success
---
```
