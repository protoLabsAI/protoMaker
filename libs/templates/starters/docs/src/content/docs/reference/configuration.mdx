All site configuration lives in `astro.config.mjs` at the project root.

## Site metadata

```js
// astro.config.mjs
export default defineConfig({
  site: "https://docs.example.com", // required for sitemap + canonical URLs
  integrations: [
    starlight({
      title: "My Project",
      description: "Documentation for My Project.",
    }),
  ],
});
```

| Option        | Type     | Description                                                                  |
| ------------- | -------- | ---------------------------------------------------------------------------- |
| `site`        | `string` | Your production URL. Required for sitemap and OG tags.                       |
| `title`       | `string` | Site name shown in the header and browser tab.                               |
| `description` | `string` | Default `<meta name="description">` for pages without their own description. |

## Sidebar

The sidebar is configured in `starlight({ sidebar: [...] })`. Each entry can be an autogenerated group (mirrors a directory) or a manual list of links.

### Autogenerate from directory

```js
sidebar: [
  { label: 'Getting Started', autogenerate: { directory: 'tutorials' } },
  { label: 'How-to Guides',   autogenerate: { directory: 'guides' } },
  { label: 'Reference',       autogenerate: { directory: 'reference' } },
],
```

### Manual links

```js
sidebar: [
  {
    label: 'Start here',
    items: [
      { label: 'Introduction', link: '/tutorials/getting-started/' },
      { label: 'Quick start',  link: '/tutorials/quick-start/' },
    ],
  },
],
```

## Social links

```js
social: [
  { icon: 'github',   label: 'GitHub',   href: 'https://github.com/your-org' },
  { icon: 'twitter',  label: 'Twitter',  href: 'https://twitter.com/your-handle' },
  { icon: 'discord',  label: 'Discord',  href: 'https://discord.gg/your-server' },
],
```

Available icons: `github`, `gitlab`, `twitter`, `mastodon`, `discord`, `slack`, `youtube`, `linkedin`, `rss`, `email`, `external`.

## Custom CSS

```js
starlight({
  customCss: ['./src/styles/global.css'],
}),
```

The starter ships `src/styles/global.css` with:

- Geist font imports
- Starlight CSS variable overrides (violet accent, dark surfaces)
- `gradient-text` utility class

## Pagefind search

Pagefind is enabled by default — no configuration required.
The search index is built automatically during `npm run build`.

To disable:

```js
starlight({
  pagefind: false,
}),
```

## Sitemap

The `@astrojs/sitemap` integration generates `/sitemap-index.xml` and `/sitemap-0.xml`
based on every page in your site. Make sure `site` is set in `defineConfig` for accurate absolute URLs.

```js
import sitemap from "@astrojs/sitemap";

export default defineConfig({
  site: "https://docs.example.com",
  integrations: [
    starlight({
      /* … */
    }),
    sitemap(),
  ],
});
```

## Cloudflare deployment

### Static (recommended for docs)

The default `output: 'static'` produces a plain `dist/` folder you can deploy directly to Cloudflare Pages:

1. Connect your Git repo in the Cloudflare dashboard
2. Set **Build command**: `npm run build`
3. Set **Build output directory**: `dist`

No additional adapter or configuration needed.

### SSR / Edge Functions

To run server-rendered routes on Cloudflare Workers:

```js
import cloudflare from "@astrojs/cloudflare";

export default defineConfig({
  output: "server", // or 'hybrid'
  adapter: cloudflare(),
  integrations: [
    starlight({
      /* … */
    }),
    sitemap(),
  ],
});
```

<Aside type="note">
  The `@astrojs/cloudflare` package is already listed in `devDependencies`. Run
  `npm install` then update the config above.
</Aside>

## ExpressiveCode (syntax highlighting)

Starlight uses [ExpressiveCode](https://expressive-code.com/) for code blocks. Change the themes:

```js
starlight({
  expressiveCode: {
    themes: ['github-dark', 'github-light'],
  },
}),
```
