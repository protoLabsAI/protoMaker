This tutorial walks you through setting up your documentation site from scratch.
By the end, you'll have a running site with live content and search.

## What you'll learn

- How to install dependencies and start the dev server
- How to create your first documentation page
- How the sidebar is automatically generated from your content

## Prerequisites

- Node.js 22 or later
- A terminal and a text editor (VS Code recommended)
- Basic familiarity with Markdown

---

## Set up your site

<Steps>

1. **Clone the repository**

   ```bash
   git clone <your-repo-url>
   cd <your-repo>
   ```

2. **Install dependencies**

   ```bash
   npm install
   ```

3. **Start the dev server**

   ```bash
   npm run dev
   ```

   Open [localhost:4321](http://localhost:4321) in your browser. You'll see the welcome page.

4. **Create your first page**

   Add a new file at `src/content/docs/guides/my-guide.mdx`:

   ```mdx
   ---
   title: My First Guide
   description: A quick guide to get you started.
   ---

   Write your content here using **Markdown**.
   ```

   The file appears automatically in the sidebar under **How-to Guides**. Refresh the browser to see it.

5. **Build for production**

   ```bash
   npm run build
   ```

   The static site is written to `dist/`. Upload it to any static host (Cloudflare Pages, Netlify, Vercel, etc.).

</Steps>

## Project structure

After setup, your project looks like this:

```
.
├── src/
│   ├── content/
│   │   └── docs/       ← your Markdown/MDX pages live here
│   │       ├── index.mdx
│   │       ├── tutorials/
│   │       ├── guides/
│   │       └── reference/
│   └── styles/
│       └── global.css  ← brand theme and Starlight overrides
├── astro.config.mjs    ← Starlight configuration
└── package.json
```

<Aside type="tip">
  The sidebar in `astro.config.mjs` uses `autogenerate` to mirror the directory
  structure inside `src/content/docs/`. No manual list to maintain — just create
  files and they appear.
</Aside>
